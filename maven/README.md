# Qentrix Selenium TestNG Framework

## Project Overview
This is a Selenium-based test automation framework using Java and TestNG for the Qentrix project.

## Prerequisites
- Java 11 or higher
- Maven 3.6 or higher
- Chrome/Firefox/Edge browser installed

## Setup Instructions

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd maven
   ```

2. **Install dependencies**
   ```bash
   mvn clean install
   ```

3. **Run tests**
   ```bash
   mvn test
   ```

## Running Tests

### Run all tests
```bash
mvn test
```

### Run specific test suite
```bash
mvn test -DsuiteXmlFile=testng.xml
```

### Run with specific browser
```bash
mvn test -Dbrowser=chrome
mvn test -Dbrowser=firefox
mvn test -Dbrowser=edge
```

## Page Object Model

The framework follows the Page Object Model (POM) design pattern:

- **LoginPage**: Contains methods for login functionality
- **LeavePage**: Contains methods for leave management operations

## Test Base Class

`BaseTest.java` provides:
- WebDriver initialization and configuration
- Browser setup with WebDriverManager
- Teardown functionality
- Common test utilities

## Reporting

TestNG generates HTML reports in the `target/surefire-reports` directory after test execution.

## Notes

- Tests use WebDriverManager for automatic driver management
- Page Factory pattern is used for element initialization
- Explicit waits are implemented for better synchronization
- Tests are independent and can run in any order
- Framework supports parallel execution (currently set to sequential)

## Troubleshooting

1. **Driver issues**: WebDriverManager automatically downloads required drivers
2. **Element not found**: Increase wait times in page objects if needed
3. **Browser compatibility**: Ensure browser version is compatible with Selenium version
